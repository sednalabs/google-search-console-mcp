# Linux CI Artifact Verification

The `Linux Artifact` workflow builds an evaluation-only Linux CI artifact from
`refs/heads/main`. It does not create a tag, GitHub release, package, or formal release asset.

Each artifact ZIP contains:

- `google-search-console-mcp`, the staged Linux x86_64 binary
- `google-search-console-mcp-linux-x86_64.spdx.json`, an SPDX 2.3 SBOM generated by Syft
  `v1.46.0` from the exact staged binary, whose runtime dependency graph is embedded by
  cargo-auditable `v0.7.5`
- `MANIFEST.txt`, the repository, source ref, source commit, generator version, and SHA-256
  checksums for the binary and SBOM

The workflow run summary records the immutable Actions artifact ID, URL, and ZIP digest. The
main-only attestation job downloads that same-run artifact by ID, verifies the ZIP digest, and
creates both SLSA provenance and SPDX SBOM attestations for that exact ZIP digest.

SBOM generation fails closed unless the SPDX document contains at least two Rust packages and at
least one `DEPENDENCY_OF` relationship. The package and dependency-relationship counts are also
recorded in `MANIFEST.txt`; a structurally valid but empty SBOM is not published.

## Verify An Artifact

Set `RUN_ID` to a successful `Linux Artifact` run on `main`, then download the original artifact
ZIP without changing its bytes:

```bash
REPOSITORY=sednalabs/google-search-console-mcp
RUN_ID=<main-workflow-run-id>
ARTIFACT_NAME=google-search-console-mcp-linux-x86_64
ARTIFACT_ZIP="${ARTIFACT_NAME}.zip"

ARTIFACT_ID="$(
  gh api "repos/${REPOSITORY}/actions/runs/${RUN_ID}/artifacts" \
    --jq ".artifacts[] | select(.name == \"${ARTIFACT_NAME}\") | .id"
)"
test -n "${ARTIFACT_ID}"
gh api "repos/${REPOSITORY}/actions/artifacts/${ARTIFACT_ID}/zip" > "${ARTIFACT_ZIP}"
```

Verify provenance while binding trust to this repository, this signer workflow, protected main,
and GitHub-hosted runners:

```bash
gh attestation verify "${ARTIFACT_ZIP}" \
  --repo sednalabs/google-search-console-mcp \
  --signer-workflow sednalabs/google-search-console-mcp/.github/workflows/linux-artifact.yml \
  --source-ref refs/heads/main \
  --deny-self-hosted-runners
```

Verify the SPDX 2.3 SBOM attestation under the same identity constraints:

```bash
gh attestation verify "${ARTIFACT_ZIP}" \
  --repo sednalabs/google-search-console-mcp \
  --signer-workflow sednalabs/google-search-console-mcp/.github/workflows/linux-artifact.yml \
  --source-ref refs/heads/main \
  --deny-self-hosted-runners \
  --predicate-type https://spdx.dev/Document/v2.3
```

Finally, verify the checksums embedded inside the CI artifact:

```bash
ARTIFACT_DIR="$(mktemp -d)"
unzip -q "${ARTIFACT_ZIP}" -d "${ARTIFACT_DIR}"
(
  cd "${ARTIFACT_DIR}"
  sha256sum --check --strict MANIFEST.txt
)
```
